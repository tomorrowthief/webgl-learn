## 1. 背景与目标

### 1.1 项目背景

WebGL 和 Three.js 是前端 3D 可视化领域的重要技术栈，广泛应用于数据可视化、游戏开发、产品展示、数字孪生等场景。但目前市面上的学习资源以文档阅读和视频教程为主，缺少一个可**边看效果、边改代码、即时运行**的在线交互式学习平台。

本项目旨在构建一个纯前端、零依赖后端的 WebGL/Three.js 在线学习系统，用户无需安装任何 IDE 或本地环境，打开浏览器即可上手学习。

### 1.2 产品目标

| 目标 | 衡量指标 |
|------|----------|
| 提供即开即用的 3D 编程体验 | 用户打开浏览器即可看到运行效果，无需安装 |
| 降低学习门槛 | 预置案例模板，用户从修改代码开始而非从零搭建 |
| 即时反馈 | 代码编辑后点击运行，1 秒内看到结果 |
| 覆盖初级到高级学习路径 | 至少提供 8-12 个分级案例（初/高各半） |
| 纯前端可部署 | 产物为静态文件，可托管于任何 CDN / GitHub Pages |

### 1.3 目标用户

- 前端开发者想入门 Three.js / WebGL
- 计算机专业学生做图形学实验
- 设计师/创意开发者探索 3D 网页效果
- 任何对 3D 编程感兴趣的互联网用户

## 2. 功能清单

---

### F1 首页 — 案例卡片展示

**描述：**
首页以卡片网格形式展示所有可用的学习案例。每张卡片包含案例缩略图（静态截图/封面）、案例标题、简短描述、难度标签（初级/高级）。

**用户故事：**
> 作为学习者，我希望在首页浏览所有案例，快速了解每个案例的内容和难度，以便选择适合我的案例。

**验收标准：**

- [ ] **AC1.1** 页面加载后 1 秒内完成卡片渲染（首屏 LCP < 1.5s）
- [ ] **AC1.2** 每张卡片显示：缩略图、标题（≤ 20 字）、简述（≤ 60 字）、难度标签
- [ ] **AC1.3** 卡片按默认排序展示（难度从低到高）
- [ ] **AC1.4** 卡片数量 ≥ 8 张（初级 ≥ 4，高级 ≥ 4）
- [ ] **AC1.5** 卡片为响应式网格布局：桌面 3-4 列，平板 2 列，手机 1 列
- [ ] **AC1.6** 卡片具有 hover 动效（微放大 + 阴影增强），点击跳转至详情页
- [ ] **AC1.7** 卡片区域支持键盘导航（Tab 聚焦，Enter 跳转）

**优先级：** P0 — 核心功能

---

### F2 首页 — 分类标签切换（初级 / 高级）

**描述：**
首页顶部提供两个 Tab 标签：「初级案例」和「高级案例」。默认展示「全部案例」（或「初级案例」），点击后按所选分类过滤卡片。

**用户故事：**
> 作为学习者，我希望按难度筛选案例，只看初级或只看高级，避免被不适合我的内容干扰。

**验收标准：**

- [ ] **AC2.1** 默认展示全部案例，Tab 状态为"全部"选中
- [ ] **AC2.2** 提供三个 Tab：全部、初级案例、高级案例
- [ ] **AC2.3** 切换 Tab 时卡片过滤动画平滑（fade-in/out，300ms 内完成）
- [ ] **AC2.4** 当前选中 Tab 有明确的视觉高亮（颜色、下划线等）
- [ ] **AC2.5** 空状态处理：若某分类无案例，显示占位提示（如"暂无案例" + 插图）
- [ ] **AC2.6** Tab 切换时 URL hash 同步更新（如 `#/beginner`），支持浏览器前进/后退

**优先级：** P0 — 核心功能

---

### F3 详情页 — 3D 实时渲染面板

**描述：**
详情页左侧区域为 Three.js 3D 渲染画布，实时显示当前代码的运行效果。画布采用标准 WebGL 渲染，支持用户旋转、缩放、平移等基础相机交互。

**用户故事：**
> 作为学习者，我希望在编辑代码后能在左侧直接看到 3D 效果，立即验证我的修改是否正确。

**验收标准：**

- [ ] **AC3.1** 左侧渲染区域占页面宽度 ≥ 45%（桌面端），采用 `<canvas>` 元素渲染
- [ ] **AC3.2** 渲染画布自适应容器大小，窗口 resize 时重新渲染
- [ ] **AC3.3** 默认支持 OrbitControls（鼠标旋转、滚轮缩放、右键平移）
- [ ] **AC3.4** 初次加载时显示位于左侧的 loading 占位（或骨架屏），代码运行后显示 3D 内容
- [ ] **AC3.5** 画布背景色与页面协调（深色系，如 #1a1a2e）
- [ ] **AC3.6** 渲染帧率 ≥ 30fps（中低端设备）
- [ ] **AC3.7** 渲染区域为空时显示提示："点击右侧「运行」按钮查看效果"
- [ ] **AC3.8** 运行时如发生 WebGL 上下文丢失，自动尝试恢复并提示用户

**优先级：** P0 — 核心功能

---

### F4 详情页 — 代码编辑器

**描述：**
详情页右侧区域为在线代码编辑器，支持 JavaScript/GLSL 语法高亮、行号显示、自动缩进、括号匹配等基本编辑功能。编辑器预填充当前案例的模板代码。

**用户故事：**
> 作为学习者，我希望在浏览器中有一个好用的代码编辑器，有语法高亮和行号，方便我阅读和修改代码。

**验收标准：**

- [ ] **AC4.1** 右侧编辑器占页面宽度约 50%-55%（桌面端）
- [ ] **AC4.2** 使用 Monaco Editor（VS Code 内核）或 CodeMirror，提供：
  - JavaScript 语法高亮
  - 行号显示
  - 自动缩进
  - 括号/花括号匹配高亮
  - 代码折叠（可选，P1）
- [ ] **AC4.3** 编辑器主题为深色系（如 `vs-dark` 或自定义深色主题），与画布风格统一
- [ ] **AC4.4** 编辑器字体为等宽字体（如 JetBrains Mono / Fira Code），字号 ≥ 14px
- [ ] **AC4.5** 编辑器高度自适应页面，不出现横向滚动条（自动换行）
- [ ] **AC4.6** 编辑器支持常用快捷键：
  - `Ctrl/Cmd + S`：运行代码
  - `Ctrl/Cmd + Z / Shift + Z`：撤销/重做
  - `Tab / Shift + Tab`：缩进/反缩进
- [ ] **AC4.7** 移动端编辑器提供最小可用体验（语法高亮 + 基本编辑），可降级为 textarea + 模拟行号

**优先级：** P0 — 核心功能

---

### F5 详情页 — 运行按钮与代码执行

**描述：**
代码编辑区域上方（或下方）有一个「运行」按钮。用户编辑代码后点击运行按钮，左侧渲染面板会销毁旧的 Three.js 场景并执行新代码，展示最新效果。

**用户故事：**
> 作为学习者，我希望修改代码后点击运行按钮就能看到效果，不需要刷新整个页面。

**验收标准：**

- [ ] **AC5.1** 运行按钮位置明显，位于编辑器工具栏区域，样式为主色调醒目按钮
- [ ] **AC5.2** 点击运行后，执行流程：
  1. 销毁旧场景（dispose geometry/material/texture/renderer）
  2. 创建新的渲染上下文
  3. 执行用户代码
  4. 渲染到左侧画布
- [ ] **AC5.3** 运行过程有视觉反馈：按钮显示 loading 状态（spinner 或文字变化为"运行中…"），执行完成后恢复
- [ ] **AC5.4** 代码执行时间 > 5 秒时，自动终止并提示超时
- [ ] **AC5.5** 快捷键 `Ctrl/Cmd + Enter` 触发热运行（hot-run）
- [ ] **AC5.6** 运行失败时，错误信息以友好格式显示在渲染区域（红色边框 + 错误文本），而非浏览器 console 报错
- [ ] **AC5.7** 连续点击运行时，前一次运行如未完成则取消前一次（幂等运行）

**优先级：** P0 — 核心功能

---

### F6 详情页 — 模板代码预填充

**描述：**
用户从首页进入某个案例详情页时，右侧编辑器自动填充该案例的模板代码（包含 Three.js 初始化、场景搭建的基本框架），左侧可能已经自动运行一次展示默认效果。同时提供「重置」按钮恢复为初始模板。

**用户故事：**
> 作为学习者，我希望进入案例后编辑器已经有初始代码，我可以在此基础上修改学习，而不是从零写起。

**验收标准：**

- [ ] **AC6.1** 进入详情页后，Monaco Editor 自动加载并显示该案例的模板代码
- [ ] **AC6.2** 模板代码至少包含三项必要内容：
  1. `import * as THREE from 'three'` 或 CDN 依赖声明
  2. 场景初始化（scene, camera, renderer）
  3. 案例核心逻辑（如创建几何体、添加光源等）
- [ ] **AC6.3** 页面加载完成后自动运行一次模板代码，展示初始 3D 效果
- [ ] **AC6.4** 提供「重置代码」按钮，点击后编辑器恢复为模板代码并自动运行
- [ ] **AC6.5** 重置操作有二次确认提示："确定要重置代码吗？您的修改将丢失。"
- [ ] **AC6.6** 模板代码文件以独立 `.js` 文件存放在项目 `src/examples/` 目录下，便于维护

**优先级：** P0 — 核心功能

### F7 错误提示与安全沙箱

**描述：**
用户编写的代码在隔离的沙箱环境中执行，防止恶意代码影响页面安全。语法错误和运行时错误以友好的方式提示用户，并标注错误行号。

**用户故事：**
> 作为学习者，当我代码写错时，希望能看到清晰的错误提示告诉我哪里错了，同时不用担心写错代码会搞崩整个页面。

**验收标准：**

- [ ] **AC7.1** 代码在 `<iframe>` sandbox 或 Web Worker 中执行，与主页面隔离
- [ ] **AC7.2** 错误信息包含：错误类型（SyntaxError / ReferenceError / TypeError 等）、错误消息、错误行号
- [ ] **AC7.3** 错误信息在渲染区域以红色提示框展示，同时在编辑器中高亮错误行（红色波浪下划线）
- [ ] **AC7.4** 沙箱禁止访问 `document`、`window`、`localStorage`、`fetch` 等危险 API（或做白名单限制）
- [ ] **AC7.5** 无限循环保护：执行超过 5 秒自动中断
- [ ] **AC7.6** 内存保护：单个案例渲染内存占用 < 512MB，超出时提示用户
- [ ] **AC7.7** 错误提示支持中英文（当前阶段中文优先）

**优先级：** P1 — 重要功能

### F8 首页 — 案例搜索与筛选

**描述：**
首页顶部提供搜索框，用户输入关键词可实时过滤案例卡片。搜索结果高亮匹配文字。

**用户故事：**
> 作为学习者，当案例数量增多时，我希望通过搜索快速找到我感兴趣的案例。

**验收标准：**

- [ ] **AC8.1** 搜索框位于首页顶部，Tab 下方
- [ ] **AC8.2** 输入关键词后实时过滤（debounce 300ms），无需点击搜索按钮
- [ ] **AC8.3** 搜索范围：案例标题 + 描述文字
- [ ] **AC8.4** 搜索结果中匹配文字高亮显示（如黄色背景）
- [ ] **AC8.5** 无匹配结果时显示空状态："未找到相关案例"
- [ ] **AC8.6** 搜索与分类 Tab 联动：选择"初级"后搜索只在初级案例中过滤
- [ ] **AC8.7** 搜索框支持一键清除（× 按钮或 Esc 键）

**优先级：** P1 — 增强功能

### F9 响应式布局适配

**描述：**
系统在桌面端、平板端、手机端均有良好的浏览和交互体验。详情页在移动端采用上下布局（3D 效果在上，编辑器在下）替代左右布局。

**用户故事：**
> 作为学习者，我希望在手机或平板上也能浏览案例和学习代码，方便我随时学习。

**验收标准：**

- [ ] **AC9.1** 断点设计：
  - 桌面端（≥ 1024px）：首页 3 列卡片，详情页左右布局
  - 平板端（768px-1023px）：首页 2 列卡片，详情页可保持左右布局（编辑器缩窄）
  - 手机端（< 768px）：首页 1 列卡片，详情页上下布局（渲染画布在上，编辑器在下）
- [ ] **AC9.2** 手机端详情页渲染画布高度 ≥ 40vh
- [ ] **AC9.3** 所有触摸交互正常（OrbitControls 支持单指旋转、双指缩放/平移）
- [ ] **AC9.4** 导航栏在移动端收缩为汉堡菜单（如有导航）
- [ ] **AC9.5** 字体大小适配：桌面 ≥ 14px，手机 ≥ 16px（防止 iOS 缩放）
- [ ] **AC9.6** 所有按钮触摸目标 ≥ 44x44px（符合 WCAG 标准）

**优先级：** P0 — 核心体验

### F10 浏览器兼容性处理

**描述：**
系统在主流浏览器上正常运行。对不支持 WebGL 的浏览器提供降级提示。

**用户故事：**
> 作为学习者，我希望在 Chrome、Edge、Firefox、Safari 等常用浏览器中都能正常访问学习。

**验收标准：**

- [ ] **AC10.1** 支持浏览器：Chrome 90+、Edge 90+、Firefox 90+、Safari 15+
- [ ] **AC10.2** 页面加载时检测 WebGL 支持，如不支持则显示友好提示：
  > "您的浏览器不支持 WebGL，请使用最新版 Chrome/Edge/Firefox/Safari 访问"
- [ ] **AC10.3** WebGL 2.0 优先，自动降级到 WebGL 1.0
- [ ] **AC10.4** iOS Safari 上 WKWebView 的 WebGL 兼容处理（抗锯齿可能不可用）
- [ ] **AC10.5** 各浏览器下 Monaco Editor 正常加载和编辑
- [ ] **AC10.6** CSS Grid / Flexbox 兼容处理（必要时使用 autoprefixer）

**优先级：** P1 — 基础能力

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标值 |
|------|--------|
| 首屏加载（LCP） | < 1.5s（桌面端） |
| 首次交互（FID） | < 100ms |
| 首页到详情页跳转 | < 500ms（SPA 路由切换） |
| 代码运行到画面渲染 | < 1s（模板代码） |
| 渲染帧率 | ≥ 30fps（中端设备） |
| 构建产物大小（gzip） | < 500KB（不含 Three.js CDN） |

### 4.2 可用性

- 所有交互需提供视觉反馈（hover、active、loading 状态）
- 错误信息使用用户可理解的语言（非技术错误码）
- 支持键盘快捷操作（运行、重置等）
- WCAG 2.1 AA 级别色彩对比度

### 4.3 可维护性

- 案例模板代码独立存放（`src/examples/` 目录），每个案例一个 `.js` 文件
- 案例元数据（标题、描述、难度、缩略图、模板文件路径）集中定义在 `src/examples/index.js` 或 JSON 配置文件中
- 组件化架构：卡片组件、编辑器组件、渲染画布组件独立封装
- 代码规范：ESLint + Prettier

### 4.4 可部署性

- 纯前端项目，构建产物为静态文件（HTML + JS + CSS + assets）
- 支持 `npm run build` 一键构建
- 支持 `npm run dev` 本地开发热更新
- 可部署至任意静态托管（GitHub Pages / Vercel / Netlify / CDN）

### 4.5 安全性

- 用户代码在 iframe sandbox 中执行，不允许 `allow-same-origin`
- 不允许使用 `eval()` / `new Function()` 执行任意代码（模板代码编译/转译方式需安全审查）
- XSS 防护：用户输入（搜索结果等）做 HTML 转义
- CSP 策略：不加载外部不可信脚本

- 开发环境：通过 importmap 或 CDN script 标签引入
- 用户模板代码：使用 `import * as THREE from 'https://...'` importmap 方式确保与沙箱兼容
